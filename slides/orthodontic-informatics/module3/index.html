Proprietary-based Interoperability

  • Custom APIs made available by Vendors
  • Documentation and implementation availability at discretion of vendor
  • Additional software that acts as bridges between these custom APIs needs to be developed
  • Single implementation allows only two vendors to link together
2 individuals can work with each other in various ways: they can come up with their own way of collaborating, or they can use a commonly known way to collaborate.

Proprietary-based Interoperability

  • Expensive solutions
  • When vendor upgrades or changes their product, the API often changes and the bridge brakes
  • Medical provider cannot see what's going on
  • Not sustainable solutions

Open Standards-based Interoperability

  • DICOM/HL7/FHIR: long-lasting standards
  • Documentation and implementation tools available online to everyone
  • Single implementation allows multiple vendor products to link together
  • Less Expensive solutions

Open Standards-based Interoperability

  • When vendor upgrades or changes their software, they keep DICOM/HL7 part the same
  • Medical provider has the opportunity to see what's going on
  • Sustainable solution

Standards Development Process

  • Functional Profiles
  • Standards improvements/corrections
  • Implementation Guidelines/Profiles
  • Testing (Connectathons)
  • The doctors (aka domain experts) define their needs into Functional Profiles
  • The standards experts analyze (gap analysis) if the standard in question is capable of fulfilling the specific functional requirement. If something is missing, a standard correction/improvement is worked on.
  • Step by step Guidelines for developers are written and published. These are instructions that a developer can use to make sure their software implements the standard properly.
  • Vendors get together and test their implementations.
  • When enough implementation and testing has been done, the standard becomes established.

What is going on

  • 1985: DICOM a standard for medical imaging
  • 1989: HL7v2 a standard for medical clinical messages
  • 1992: GEHR: the good European health record
  • 1999: IHE First Connectathon
  • 1999: ADA forms the Standards Committee on Dental Informatics
  • 2003: openEHR
  • 2005: HL7v3
  • 2014: HL7 FHIR
  • I don't know all the history of what happened when, I will research this better in the future, but I do know enough to give you an idea.
